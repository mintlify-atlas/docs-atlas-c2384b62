## Overview

This guide walks you through the complete user flow of the UCC Control de Acceso system, from initial login to reporting a card failure. The entire process takes less than 2 minutes.

<Note>
**Before you start**: This system is only for users who do NOT have their physical TIC (Tarjeta de Identificación Cooperativista) at the time of entry.
</Note>

## Prerequisites

You'll need:

- Your **Institutional ID** (e.g., 80123456)
- Access to the system URL (provided by your campus security)
- A web browser (works on mobile and desktop)

## Step 1: Access the Login Page

Navigate to the UCC Control de Acceso application. You'll see a clean login screen with the UCC logo.

<Note>
The login page features UCC institutional branding with blue and orange colors.
</Note>

The login page features:
- UCC institutional branding (blue and orange colors)
- Single input field for your institutional ID
- Clear instructions that this is for contingency access only

## Step 2: Enter Your Institutional ID

<Steps>
  <Step title="Input your ID">
    Type your institutional ID in the input field (e.g., `80123456`)
    
    ```jsx
    // The system validates that only numbers are entered
    inputMode="numeric"
    pattern="\d*"
    ```
    
    <Note>
    The field only accepts numeric input for security and validation purposes.
    </Note>
  </Step>
  
  <Step title="Submit the form">
    Click the **"INGRESAR"** button to validate your ID
    
    The system will:
    - Verify your ID exists in the database
    - Retrieve your complete user profile
    - Check your current access status (active/blocked)
  </Step>
</Steps>


<Warning>

<Tabs>

  </Tab>
  
  <Tab title="Immediate Block (Lost/Stolen)">
    <Card title="🚨 BLOQUEO INMEDIATO" icon="triangle-exclamation" color="#EF4444">

      **What to do**: 
      1. Visit the administration office
      2. Request a new TIC
      3. Request an unblock from an administrator
    </Card>
  </Tab>
</Tabs>

### Database Changes on Report

When you report a card failure, the following happens in the database:

```sql
-- For "Olvido" (Forgot)
-- 1. Insert into Registro_Fallas
INSERT INTO Registro_Fallas (
  usuario_id, 
  fecha_hora, 
  motivo, 
  sede_registro,
  vigilante_registro
) VALUES (
  1, 
  NOW(), 
  'olvido', 
  'Bogotá',
  'Juan Seguridad'
);

-- 2. Increment failure counter
UPDATE Usuarios 
SET total_fallas = total_fallas + 1,
    updated_at = NOW()
WHERE id = 1;

-- 3. If total_fallas = 4, block user
UPDATE Usuarios 
SET estado_general = 'bloqueado'
WHERE id = 1 AND total_fallas >= 4;

-- 4. Create block record
INSERT INTO Historial_Bloqueos (
  usuario_id,
  fecha_bloqueo,
  motivo_bloqueo,
  estado
) VALUES (
  1,
  NOW(),
  'limite_fallas',
  'activo'
);
```

## Understanding the Failure System

<Card title="The 4-Failure Rule" icon="chart-line">
  The system uses a progressive warning system:
  
  | Failure Count | Status | Access | Risk Level |
  |---------------|--------|--------|------------|
  | 0 | ⚫ Clean | ✅ Yes | None |
  | 1 | 🔵 First warning | ✅ Yes | Low |
  | 2 | 🟠 Second warning | ✅ Yes | Medium |
  | 3 | 🟡 Final warning | ⚠️ Yes | **High** |
  | 4 | 🔴 Blocked | ❌ No | Blocked |
</Card>

### Visual Indicators

The system provides clear visual feedback about your status:

```jsx
{/* Failure badge on profile header */}
{user.total_fallas > 0 && (
  <div className={`
    mt-3 px-4 py-1.5 rounded-full text-xs font-semibold 
    ${user.total_fallas >= 3
      ? 'bg-orange-500 text-white'  // Danger zone
      : 'bg-blue-700 text-blue-100' // Normal
    }
  `}>
    {user.total_fallas}/4 fallas registradas
  </div>
)}
```

## Complete User Flow Example

Let's walk through a real-world scenario:

<Steps>
  <Step title="María's First Failure">
    María is a student who forgot her card. She:
    1. Opens the system and enters her ID: `80234567`
    2. Sees her profile with 0 failures
    3. Clicks "Reportar TIC"
    4. Selects "Olvidé mi TIC"
    5. Sees "✅ Ingreso Permitido - 1/4 fallas"
    6. Enters campus
  </Step>
  
  <Step title="Second Time (One Week Later)">
    María forgot her card again:
    1. Logs in with ID `80234567`
    2. Now sees 1 previous failure in history
    3. Reports "Olvidé mi TIC" again
    4. Counter increases to 2/4
    5. Access still granted
  </Step>
  
  <Step title="Third Time - Warning Zone">
    María's profile now shows:
    ```jsx
    <div className="bg-orange-500 text-white">
      3/4 fallas registradas
    </div>
    ```
    
    After reporting the third time:
    - ⚠️ High risk warning displayed
    - Message: "Una falla más y serás bloqueada"
    - Access granted but with strong warning
  </Step>
  
  <Step title="Fourth Time - Blocked">
    María forgot her card a fourth time:
    1. Reports "Olvidé mi TIC"
    2. System automatically blocks her
    3. Status changes from "Activo" to "Bloqueado"
    4. Red message: "🚫 USUARIO BLOQUEADO"
    5. Must visit administration office
  </Step>
  
  <Step title="Getting Unblocked">
    María visits administration:
    1. Admin searches for her ID
    2. Admin reviews her failure history
    3. Admin clicks "Desbloquear Usuario"
    4. Enters justification note
    5. María's status returns to "Activo"
    6. Failure counter resets to 0
    7. Action recorded in audit log
  </Step>
</Steps>

## Troubleshooting

<AccordionGroup>
  <Accordion title="My ID is not recognized">
    **Possible causes**:
    - ID entered incorrectly (check for typos)
    - Not enrolled for current semester
    - Database not updated yet
    
    **Solution**: Contact campus administration to verify your enrollment status.
  </Accordion>
  
  <Accordion title="I'm blocked but it's my first time">
    **Possible causes**:
    - You may have previous failures from earlier in the semester
    - You selected "Perdí mi TIC" which causes immediate blocking
    
    **Solution**: Check your failure history on your profile. Visit administration if you believe this is an error.
  </Accordion>
  
  <Accordion title="The page won't load">
    **Possible causes**:
    - Network connectivity issues
    - System maintenance
    - Browser compatibility
    
    **Solution**: Try refreshing the page, check your internet connection, or try a different browser.
  </Accordion>
  
  <Accordion title="I accidentally selected the wrong option">
    **Solution**: Immediately contact the security desk or administration before the report is processed. They can review and correct the entry if caught quickly.
  </Accordion>
</AccordionGroup>

## Best Practices

<Card title="Tips for Avoiding Blocks" icon="lightbulb">
  1. **Always bring your physical TIC** - This system is only for emergencies
  2. **Set reminders** - Use your phone to remind you to pack your card
  3. **Keep a backup** - Consider keeping a photo of your ID on your phone (for reference, not access)
  4. **Track your failures** - Monitor your failure count in the system
  5. **Act at 3 failures** - If you reach 3/4, be extra careful as one more will block you
  6. **Report accurately** - Always select the correct reason (forgot vs. lost/stolen)
</Card>
